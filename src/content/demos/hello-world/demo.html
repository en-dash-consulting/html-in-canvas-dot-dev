Hello from HTML‑in‑Canvas!
layoutsubtree
Canvas attribute that enables child element layout and painting.
onpaint
Callback fired when child content changes and the canvas needs re-rendering.
drawElementImage()
Draws a child element into the canvas. Returns a DOMMatrix for position sync.
ResizeObserver
Keeps the canvas pixel resolution matched to its CSS layout size.

Annotated Source

<!-- 1. The HTML -->
<canvas id="canvas" layoutsubtree>
  <div id="content">Hello!</div>
</canvas>
layoutsubtree tells the browser to lay out children inside the canvas. They participate in accessibility and hit testing but are only visible when drawn.

// 2. Get references
const canvas = document.getElementById("canvas");
const ctx = canvas.getContext("2d");
const content = document.getElementById("content");

// 3. Paint callback
canvas.onpaint = () => {
  ctx.reset();
  const t = ctx.drawElementImage(content, 0, 0);
  content.style.transform = t.toString();
};
drawElementImage() composites the element onto the canvas and returns a DOMMatrix. Applying it as a CSS transform keeps the DOM position in sync with the drawn position — essential for hit testing and accessibility.

// 4. ResizeObserver for crisp rendering
new ResizeObserver(([entry]) => {
  const dp = devicePixelRatio;
  const { width, height } = entry.contentRect;
  canvas.width  = Math.round(width * dp);
  canvas.height = Math.round(height * dp);
  ctx.scale(dp, dp);
  if (canvas.onpaint) canvas.onpaint();
}).observe(canvas);
ResizeObserver keeps the canvas backing store sized to device pixels. After resizing, ctx.scale(dp, dp) adjusts the transform so drawing coordinates stay in CSS pixels.
