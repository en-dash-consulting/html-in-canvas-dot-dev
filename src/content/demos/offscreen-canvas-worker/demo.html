Main Thread
// 1. Transfer canvas control to a worker
const canvas = document.getElementById("output");
const offscreen = canvas.transferControlToOffscreen();

const worker = new Worker(workerURL);
worker.postMessage(
  { type: "init", canvas: offscreen },
  [offscreen]  // transfer list
);
OffscreenCanvas transfer. Calling transferControlToOffscreen() detaches the canvas backing store from the main thread. The worker now owns it exclusively — drawing happens off the main thread, keeping the UI responsive.

// 2. Capture HTML as a transferable ElementImage
const source = document.getElementById("source");
const image = await captureElementImage(source);
captureElementImage() takes a snapshot of a DOM element and produces an ElementImage — a lightweight, GPU-backed representation that can be transferred (zero-copy) to a worker via postMessage.

// 3. Send the ElementImage to the worker
worker.postMessage(
  { type: "image", image: image },
  [image]  // transferable — zero-copy
);
Zero-copy transfer. By listing the ElementImage in the transfer list, ownership moves to the worker without serialization. The main thread can no longer use the image after transfer.

// 4. Receive sync transform from worker
worker.onmessage = (e) => {
  if (e.data.type === "transform") {
    const m = e.data.matrix;
    source.style.transform =
      `matrix(${m.a},${m.b},${m.c},${m.d},${m.e},${m.f})`;
  }
};
Transform sync. The worker sends back the DOMMatrix it used to draw the element. The main thread applies this as a CSS transform so hit testing and accessibility remain in sync with the visual position.
Worker Thread
// Worker: receive OffscreenCanvas + ElementImage
let canvas, ctx, currentImage;

self.onmessage = (e) => {
  const msg = e.data;

  if (msg.type === "init") {
    canvas = msg.canvas;
    ctx = canvas.getContext("2d");
    tick();
  }

  if (msg.type === "image") {
    currentImage = msg.image;
  }
};
Message dispatch. The worker handles two message types: init delivers the OffscreenCanvas (transferred, not copied), and image delivers a new ElementImage snapshot to draw.

// Worker: animation loop
function tick() {
  ctx.clearRect(0, 0, canvas.width, canvas.height);

  if (currentImage) {
    ctx.save();
    ctx.translate(cx, cy);
    ctx.rotate(angle);
    ctx.drawImage(currentImage, -iw/2, -ih/2);
    const t = ctx.getTransform();
    ctx.restore();

    // Send transform back to main thread
    self.postMessage({
      type: "transform",
      matrix: { a: t.a, b: t.b, ... }
    });
  }

  requestAnimationFrame(tick);
}
Worker animation. The worker runs its own requestAnimationFrame loop — completely independent of the main thread. drawImage() accepts the ElementImage just like any other image source. The transform matrix is sent back each frame for accessibility sync.
