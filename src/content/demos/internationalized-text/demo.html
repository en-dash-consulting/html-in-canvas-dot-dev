Why This Matters

  • fillText() draws glyphs in one direction with no access to the browser's text shaping, bidi, or layout engine.
  • drawElementImage() renders the browser's own HTML/CSS output — inheriting full Unicode support, writing modes, and shaping.
  • Ruby annotations, vertical text, and complex scripts work with zero extra code — just HTML and CSS.
  • Canvas i18n is solved permanently: if the browser can render it, canvas can draw it.
