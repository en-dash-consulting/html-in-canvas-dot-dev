<!-- 1. Canvas with form children -->
<canvas id="canvas" layoutsubtree>
  <div id="form-root">
    <input type="text" placeholder="..." />
    <select>...</select>
    <input type="range" />
    <input type="checkbox" />
    <button>Submit</button>
  </div>
</canvas>
layoutsubtree turns these into real, interactive DOM elements laid out inside the canvas. Every form control works natively — typing, clicking, tabbing, and keyboard shortcuts are all preserved.

// 2. Setup
const canvas = document.getElementById("canvas");
const ctx = canvas.getContext("2d");
const form = document.getElementById("form-root");

// 3. Paint callback
canvas.onpaint = () => {
  ctx.reset();
  const t = ctx.drawElementImage(form, 0, 0);
  form.style.transform = t.toString();
  paintCount++;
};
onpaint fires whenever child content changes — including the cursor blinking in a focused text input. The DOMMatrix returned by drawElementImage() keeps DOM positions synchronized, so clicks and focus land on the correct controls.

// 4. ResizeObserver
new ResizeObserver(([entry]) => {
  const dp = devicePixelRatio;
  const { width, height } = entry.contentRect;
  canvas.width  = Math.round(width * dp);
  canvas.height = Math.round(height * dp);
  ctx.scale(dp, dp);
  if (canvas.onpaint) canvas.onpaint();
}).observe(canvas);
No special event wiring needed. The browser routes all pointer, keyboard, and focus events to the child elements. The onpaint callback handles the visual updates automatically.
Native interactivity
Form controls inside a layoutsubtree canvas receive clicks, focus, keyboard input, and tab navigation with zero extra code.
Cursor blink = paint
A blinking text cursor changes the visual output, so the browser fires onpaint on each blink — watch the counter climb.
DOMMatrix sync
drawElementImage() returns a DOMMatrix. Applying it as a CSS transform keeps hit testing accurate for every control.
Accessibility
Because these are real DOM elements, screen readers, autofill, and form validation all work as expected.
