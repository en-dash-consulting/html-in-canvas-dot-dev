// 2. Element state: position, rotation, scale
const elements = [
  { el, x: 30,  y: 20,  rotate: 0,   scale: 1   },
  { el, x: 240, y: 30,  rotate: 0,   scale: 0.9 },
  { el, x: 160, y: 160, rotate: 0,   scale: 1   },
  { el, x: 50,  y: 200, rotate: -5,  scale: 1   },
  { el, x: 280, y: 250, rotate: 0,   scale: 1   },
];
Composition state — each element tracks its own x/y position, rotation angle, and scale factor. The array order defines z-stacking: first drawn = furthest back.

// 3. Paint callback with changedElements
canvas.onpaint = (event) => {
  ctx.reset();

  // Draw a subtle grid background
  drawGrid(ctx, w, h);

  // Draw each element in z-order
  for (const item of elements) {
    ctx.save();
    ctx.translate(item.x, item.y);
    ctx.rotate(item.rotate * Math.PI / 180);
    ctx.scale(item.scale, item.scale);
    const t = ctx.drawElementImage(item.el, 0, 0);
    item.el.style.transform = t.toString();
    ctx.restore();
  }
};
Z-ordering by draw order. Elements drawn later appear on top. Each element gets its own save/translate/rotate/scale/restore block. The DOMMatrix from drawElementImage() syncs hit testing for every element independently.

// 4. Drag handling
canvas.addEventListener("pointerdown", (e) => {
  // Hit test: find topmost element
  const hit = hitTest(e.offsetX, e.offsetY);
  if (!hit) return;

  // Move to front (end of array)
  elements.splice(elements.indexOf(hit), 1);
  elements.push(hit);

  // Track drag offset
  const startX = e.offsetX - hit.x;
  const startY = e.offsetY - hit.y;

  const onMove = (ev) => {
    hit.x = ev.offsetX - startX;
    hit.y = ev.offsetY - startY;
    canvas.requestPaint();
  };
  ...
});
Drag-to-reposition. On pointer down we find the topmost element at that coordinate, move it to the end of the draw array (bringing it to front), and update its position on every pointer move. requestPaint() triggers a new paint pass.
Z-ordering
Drawing order determines stacking. Elements drawn last appear on top. Reorder the array to change z-index.
changedElements
The onpaint event can report which elements changed, enabling selective repaint optimizations.
Per-element transforms
Each element gets its own save/restore block with independent translate, rotate, and scale.
requestPaint()
Called during drag to trigger a repaint after updating an element's position. The browser batches paint requests efficiently.
